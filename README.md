# rp-prompt-auditor（手術刀）

### *你的角色卡沒有人類讀者。那些寫給人看的漂亮字，模型全當雜訊收費。*

角色卡 / system prompt 的讀者只有一個：LLM。它不欣賞文筆，它按 token 計費，而且死板的模型會把你隨手寫的形容詞當成必須執行的規則。

**手術刀**是一個 token 效率審核 skill，做兩件事：

1. **找出模型不需要的字**——重複、模糊、不約束行為的鋪陳，逐條標記給你看
2. **把行為重整成穩定的 IF→THEN 結構**——讓角色可預測，而不是看模型心情

兩者衝突時，省 token 優先。

它**不做**品質評分、文筆評鑑、劇情建議。這是效率審核，不是品味審核——卡好不好看是另一回事，這裡只管卡肥不肥、穩不穩。

---

## 它不會直接改你的卡

這是刻意的設計，不是功能限制。流程強制兩段：

**Phase 1 — 審核（標記，不動刀）**

你貼卡，它出一份報告：哪些字該切、哪些必須留、跨層規則有沒有互相打架、有哪些具體的不穩定點。**這一步你的卡一個字都不會被動。**

**Phase 2 — 大砍（你拍板後才執行）**

你看完報告，告訴它：哪些不砍、哪些一起砍、哪些判定你不同意。它才依你的決定輸出改寫版，並重跑一次矛盾檢查，確認重寫沒有製造新矛盾。

你保留的內容照留，不再爭。你推翻的判定照辦——但若會引入不穩定，改寫版下方會標一行風險（位置＋後果），標完閉嘴。

改稿工具最危險的不是漏改，是在你看清楚它想幹嘛之前就把原稿蓋掉。創作者對自己的卡有最終決定權，工具只有建議權。

懶得逐項挑就打 `/動刀`，一次性授權全砍。

## 30 秒上手

**Claude.ai（網頁 / 桌面 / 手機）**

1. 下載本 repo（Code → Download ZIP），解壓縮後把資料夾改名為 `rp-prompt-auditor`
2. 重新壓成 ZIP（ZIP 根層必須是資料夾本身）
3. Claude.ai → 設定 → Customize → Skills → 上傳 ZIP，開啟開關

**Claude Code**

```
git clone https://github.com/1104angu/rp-prompt-auditor.git ~/.claude/skills/rp-prompt-auditor
```

裝好後在對話打：

```
/手術刀

（貼上你的角色定義全文）
```

卡太長可以分段貼，先講一聲還沒貼完即可，它會等你貼完才審。輸入少於 50 字它會直接說內容不足，不硬審。

| 指令 | 動作 |
|------|------|
| `/手術刀` | 啟動審核。不接材料則先跑教學 |
| `/動刀` | 一次性授權大砍，跳過取捨確認 |

## 它怎麼判

### 切除標記

| 標記 | 抓什麼 |
|------|--------|
| `[REDUNDANT]` | 重複資訊、同義措辭、無行為指向的裝飾性形容詞 |
| `[VAGUE]` | LLM 無法執行的指令。自檢：兩個不同的 LLM 讀這條，會不會做出不同的事？「適當表現」「自然回應」「依情況」一律成立 |
| `[EXCESS]` | 不約束角色行為的世界觀鋪陳 |

### 保留例外

| 標記 | 保什麼 |
|------|--------|
| `[CAUSE]` | 解釋角色為何產生某行為的情緒脈絡——刪了它，某個觸發就失去可預測輸出。整段保留 |
| `[DEFAULT]` | 角色對特定觸發的預設反應。保留 |
| `[TONE]` | 純氣氛內容。壓縮但保留核心意象 |

省 token 不等於砍到只剩骨頭。CAUSE 與 TONE 用刪除測試區分：把這段刪掉後，若仍有明確的 IF→THEN 規則涵蓋同一觸發，它就是可壓縮的氣氛；若沒有，它就是行為的因，整段留。

### 穩定性評級：不打分數

只列具體不穩定點，每點強制三要素——**位置**（哪條規則）、**失效條件**（什麼輸入會炸）、**失效後果**（角色會怎麼偏）。缺一即刪，不算數。「術語可能洩漏」這種沒頭沒尾的警告不會出現在報告裡。

## 改寫階段在防什麼

這部分是手術刀跟「叫 AI 幫我精簡」最大的差異。它針對死板模型（如 DeepSeek 一類會把規則字面硬跑的模型）的已知失效模式，內建了幾條改寫紀律：

**IF→THEN 只給硬觸發。** 死板模型會把所有形式化規則無差別硬跑，連情境不該觸發時也跑。所以只有「偶爾不發生角色就崩」的行為才配寫成 IF→THEN；性格偏好收進屬性欄當傾向。把「容易害羞」寫成 `IF 見到陌生人 → THEN 臉紅結巴`，角色在已熟識、戰鬥中也會硬臉紅——那不是害羞，是機械故障。

**壓抑 / 偽裝型規則必配出口閘門。** 只寫 `IF 偽裝中 → 收力裝弱`、不寫 `IF 身份已暴露 → 偽裝失效`，字面模型手上唯一的指令就是弱，身份被識破後還會回頭演隱藏。每條「何時該弱」都必須配一條「何時該強」，會切換的狀態寫成顯式 flag——模型追得動 flag，追不動劇情推論。

**禁令不准只靠舉例。** 所有 NEVER 必須附定義加一句二元自檢，否則模型只會迴避你舉的那幾個例子，換個說法照犯。

**改寫版輸出純淨度。** 改寫版是直接貼回平台的成品：禁星號（會被當角色定義內容讀進去）、禁 ASCII 雙引號（吃 token 且會跟 JSON 打架，引用一律用「」）、HTML 用 `~~~` 框（三反引號會被內層 code 咬斷）。

## 常見問題

### 寫完直接丟手術刀就行了嗎？

不行，順序想反了。手術刀是驗收，不是代工。它會砍廢 token、會把硬觸發整理
成穩定結構，但它不會替你寫角色。一張聲線空洞的卡審完還是空洞，只是變便宜了。

寫卡是兩層工作。結構層管「什麼必須發生」：狀態、閘門、禁令，這是手術刀的
管區。自然語言層管「像誰、什麼感覺」：聲線、文風、情緒因果。這一層手術刀
只保證不誤殺，寫還是得你自己寫。

分不清某句話該進哪層，就刪掉它試試：刪完行為變得不可預測，它是結構；刪完
角色沒了聲音，它是文風。完整寫法之後會寫一篇教學。
### 它會不會把我的角色砍成空殼？

不會，這是執行規範裡的硬規則：保留角色核心人格，僅切除 token 浪費。情緒因果（CAUSE）整段保留、氣氛（TONE）壓縮但留核心意象。而且 Phase 1 只標記不動刀——你覺得哪段是角色的魂，說一聲，它照留。

### 這跟評分工具差在哪？

評分工具回答「這張卡好不好」；手術刀回答「這張卡浪費了多少 token、哪裡會不穩」。一張文筆極好的卡可能滿身冗餘，一張樸素的卡可能結構精悍——兩種工具會給出完全不同的報告，不能互相替代。

### 如果我的卡已經寫得很省呢？

它會維持原文，明確說對在哪、拿證據。不捏造缺陷，沒有的問題不發明——這也是執行規範裡的硬規則。

### 為什麼報告口吻這麼衝？

二元判決、禁止 hedge、正面評價必須附證據——這是判斷準確度要求，不是性格。「大致良好，僅需微調」這種話對創作者沒有資訊量，指名「第 3 條、什麼輸入會炸、炸了角色會怎麼偏」才有。
